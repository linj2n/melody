Spring 提供基于注解的编程模型，例如，在模型中通过@Controller与@RestController注解表达 request 映射、request 的输入和异常处理等等。这些注解 Controller有着十分灵活的方法签名，同时既不需要继承任何的基类也不需要实现任何接口即可工作。

1. 定义

在WebApplicationContext中，可以使用标准 Spring bean 定义来定义Controller bean。

@RestController是一个组合注释，这个注释组合了@Controller与@ResponseBody。与 model-and-view 渲染不同，使用该注解的控制器下的所有方法继承了@ResponseBody，直接将信息写入 response body 映射。

2. Request Mapping

@RequestMapping注解用于将 request 映射到对应的 controller 方法。可以通过该注解的以下属性来指定请求所要满足的条件：

属性 描述
value 指定请求的 URL template。
method 指定请求的 HTTP method 类型，如method = RequestMethod.POST，表示控制器/方法接受使用 POST 方法的请求
consumes 指定该控制器/方法接受请求的 Content-Type 类型，例如consumes = {"text/plain", "application/*"}，表示控制器/方法接受Content-Type=
produces 指定请求返回的内容类型，即接受处理的请求的header中的（Accept）类型中需要包含该指定类型。
params 指定request中必须包含某些参数值时，例如params="myParam=myValue"即控制器/方法接受含有myParam=myValue的请求。
